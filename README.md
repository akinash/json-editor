# JSON Config Editor

Веб-приложение для редактирования JSON-конфига заставок веб-версии.  
Проект позволяет загружать конфиг по URL, править его в визуальном интерфейсе и в JSON-редакторе, проверять структуру данных и сохранять результат обратно в файл.

## Что умеет

- загружать JSON-конфиг по URL;
- редактировать конфиг через визуальную форму;
- синхронизировать форму и JSON-редактор в реальном времени;
- валидировать структуру и подсвечивать ошибки;
- менять порядок элементов через drag-and-drop;
- генерировать CSS-градиенты из изображения;
- работать со светлой и тёмной темой;
- экспортировать итоговый JSON-файл.

## Запуск проекта

### Установка зависимостей

```bash
npm install
```

После установки автоматически выполняется `postinstall`-скрипт, который применяет workaround для Rollup/WASM на текущей среде.

### Development mode

Запуск локального dev-сервера:

```bash
npm run dev
```

По умолчанию используется Vite dev server.

### Production build

Сборка production-версии:

```bash
npm run build
```

### Preview production build

Локальный просмотр production-сборки:

```bash
npm run preview
```

### Type check

Проверка TypeScript без сборки:

```bash
npm run typecheck
```

### Testing

Unit- и integration-тесты:

```bash
npm run test
```

End-to-end тесты:

```bash
npm run e2e
```

Если браузеры Playwright ещё не установлены, сначала выполните:

```bash
npx playwright install chromium
```

## Основные сценарии работы

### 1. Загрузка конфига

Пользователь указывает URL JSON-файла, после чего приложение загружает конфиг и нормализует его в локальную модель.

### 2. Визуальное редактирование

Каждая запись конфига редактируется через отдельный UI-блок. Для записей доступны поля для desktop/mobile-версий, правил дат, изображений, градиентов и дополнительных параметров.

### 3. Редактирование raw JSON

В приложении есть JSON-редактор, который синхронизирован с визуальной формой. Можно редактировать конфиг как в интерфейсе, так и напрямую в JSON.

### 4. Валидация

При изменениях приложение проверяет данные и показывает ошибки или предупреждения, включая проблемы со структурой, дубликатами и неполными градиентами.

### 5. Экспорт

Готовый JSON можно скачать локально как файл.

## Структура проекта

```text
src/
  app/                    провайдеры приложения
  design/                 тема, токены, глобальные стили
  features/editor/        основная логика редактора
    components/           UI-компоненты редактора
    model/                парсинг, сериализация, нормализация, валидация
  shared/                 общие типы, hooks и UI-примитивы
```

## Ключевые части приложения

- `src/features/editor/EditorPage.tsx` — главная страница редактора;
- `src/features/editor/components/ObjectCard.tsx` — редактирование отдельной записи;
- `src/features/editor/components/GradientGeneratorDialog.tsx` — генератор градиентов из изображения;
- `src/features/editor/components/JsonCodeEditor.tsx` — raw JSON-редактор;
- `src/features/editor/model/configModel.ts` — парсинг, сериализация и валидация конфигурации;
- `src/design/theme.ts` и `src/design/tokens.ts` — тема и дизайн-токены.

## Особенности реализации

- единый источник истины для данных: изменения в форме и JSON синхронизируются между собой;
- разделение UI и бизнес-логики;
- дизайн построен на токенах и общей теме;
- поддерживаются светлая и тёмная тема;
- для drag-and-drop используется `dnd-kit`;
- для редактирования JSON используется Ace Editor.

## Покрытие тестами

В проекте есть несколько уровней тестов:

- unit-тесты для бизнес-логики модели конфига;
- integration-тесты для `EditorPage`, `ObjectCard` и `GradientGeneratorDialog`;
- end-to-end тесты на Playwright для основных пользовательских сценариев.

Текущие e2e-сценарии покрывают:

- основной happy path редактирования и скачивания JSON;
- независимость `Desktop` / `Mobile` настроек;
- генерацию и применение градиента из изображения.

## Полезные замечания

- Если приложение запускается на macOS и возникают проблемы с нативными зависимостями, в проект уже добавлен workaround через `@rollup/wasm-node`.
- Если dev-сервер не стартует в режиме `file://`, это ожидаемо: загрузка по URL требует `http/https` из-за CORS-ограничений браузера.
