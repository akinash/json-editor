# UI Rules

## Цель
Единый визуальный язык для редактора JSON-конфига: предсказуемые отступы, типографика, состояния и поведение компонентов.

## Источник истины
- Токены: `src/design/tokens.ts`
- Технический снимок токенов: `src/design/tokens.json`
- Тема MUI: `src/design/theme.ts`

## Spacing
- Используется только шкала: `4, 8, 12, 16, 24, 32`
- Вложенные блоки формы: минимум `12`
- Между крупными секциями: `16` и выше

## Типографика
- Заголовок экрана: `h1`
- Заголовок секции: `h3`
- Подписи полей/пояснения: `caption`
- Основной текст: `body1`

## Цвета
- Запрещены хардкод-цвета в feature-компонентах
- Цвета берутся только из `theme.palette` или токенов
- Для светлой/тёмной темы структура ключей одинакова

## Состояния
Для интерактивных элементов обязательно:
- `default`
- `hover`
- `focus-visible`
- `disabled`
- `error` (если есть валидация)

## Формы
- Для форм использовать компоненты MUI (`TextField`, `Select`, `Switch`, `Alert`)
- Инпуты и кнопки в крупных формах: округление `pill`
- Ошибки JSON-синтаксиса блокируют скачивание
- Предупреждения бизнес-валидатора не блокируют скачивание

## Композиция
- Feature-код не должен содержать стили вне `sx`/theme
- Переиспользуемые примитивы располагать в `src/shared/ui`
- JSON-модель и валидации располагать в `src/features/editor/model`

## Pull Request Checklist
- Нет новых хардкод-цветов
- Нет нестандартных отступов вне шкалы
- Компонент поддерживает обе темы
- Прописаны все состояния интерактива
- Логика сериализации JSON не сломана
